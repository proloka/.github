# Proloka

Internal GitHub organization for the development and maintenance of the Proloka ecosystem.

This organization is intended for internal use only by the Proloka development team. It contains the source code, technical documentation, and supporting resources for all Proloka services.

## Repositories

| Repository | Description |
|------------|-------------|
| `backend` | REST API, authentication, business logic, and database services. |
| `frontend` | Web application. |
| `mobile` | Flutter mobile application. |
| `documentation` | Technical documentation, API references, architecture, and development guides. |

## Development Workflow

- Create a feature branch from `dev`.
- Open a Pull Request for review.
- Merge into `dev` after approval.
- Merge `dev` into `main` for production releases.

## Branch Strategy

| Branch | Purpose |
|---------|---------|
| `main` | Production |
| `dev` | Development |
| `feature/*` | New features |
| `fix/*` | Bug fixes |
| `hotfix/*` | Production hotfixes |

## Notes

- This organization is private and intended for internal development only.
- Do not share source code, credentials, or internal documentation outside the organization.
- Follow the coding standards and development guidelines defined in the `documentation` repository.
- Keep commits, branches, and pull requests clear and descriptive.
